# 未病レーダー

未病レーダーは、東洋医学の体質分類と天気変化を組み合わせて、体調が崩れやすいタイミングとセルフケアを提案するWebアプリです。

コアメッセージ:

> 明日の崩れやすさを、今夜の整え方に変える。

このリポジトリは、開発者がiPad上でAIにコード作成を依頼し、生成されたコードをGitHubへ反映して開発しています。

---

## v7.78.20 注意時間の変化方向と水分環境UI

- 注意時間を単なる時間帯ではなく、気象負荷が最も強い連続3時間前後として明示
- 気圧変動・寒暖差では、上昇側と低下側のピークを別々に計算し、強い方の方向を時間欄のSVG矢印で表示
- 通常の気圧上昇／低下、気温上昇／低下にも同じ方向契約を適用。高温・低温など絶対環境は従来の注意SVGを維持

- 中間帯へ戻る水分変化は、反対側の環境へ変わったように誤表示せず、戻る前の湿気／乾燥側へ含める
- `湿度差`という第3のUI分類や追加アイコンは設けない
- 気圧・気温・水分環境の方向別ピークを保存し、旧予報では従来データから安全に補完
- 体調ゆらぎ度の係数、温湿度中間帯、余力、今日01:05固定、明日16:30再計算、V1ロールバックは変更なし

## v7.78.17 気圧の物理方向と体質反応の完全分離

- 気圧の上昇・低下・反転は、天気カードに出す物理情報として維持
- 張り・力み側／重だるさ側／適応疲れという体の表れ方は、体質チェック由来の反応方向から独立して決定
- 不調サイン、東洋医学的整理、ケア方針、食事、ツボ、Ekken入力、ショップ選定まで同じ反応方向を接続
- 旧辞書の`pressure_up / pressure_down`は、物理方向ではなく反応方向から選ぶ互換キーとしてだけ使用
- 通常表示は`気温上昇／気温低下`、絶対環境は`高温／低温`、極端域は`猛暑／酷暑／厳しい寒さ`として役割を分離
- 最高気温35℃・40℃、終日氷点下または最低-5℃以下には、体質別ゆらぎ度と別枠の独自気温注意を表示。公的アラートやWBGTとは明記して区別
- 体調ゆらぎ度、V2校正値、今日01:05固定・明日16:30再計算、`RADAR_FORECAST_MODEL_VERSION=v1`へのロールバックは変更なし

詳細: `docs/RADAR_PRESSURE_RESPONSE_INTEGRATION_V77817.md`
