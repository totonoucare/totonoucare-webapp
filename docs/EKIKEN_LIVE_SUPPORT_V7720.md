# v7.72.0 ケアナビAI EKIKEN・今の体調相談

## 目的

既存のAIチャットは、選択した期間の記録を深掘りする「振り返り専用」として維持する。
新たに、体調が気になった瞬間に開ける「今の体調相談」を追加し、同じキャラクターで役割だけを分ける。

```text
期間の振り返り
  選択期間の予報・実感・ケアを詳しく聞く

今の体調相談
  今日の体調や迷いを話し、今できることを整理する
```

キャラクター名は `ケアナビAI：EKIKEN（エキケン）`。
貝原益軒の名に着想を得つつ、歴史人物を再現するAIではなく、未病レーダーの体質・予報・記録を案内するキャラクターとして扱う。

## 画面導線

- ボトムタブに `記録・相談` を追加
- 記録・相談ページは `記録 / AI分析 / 相談` の3タブ
- 相談タブ上部にEKIKENの今の体調相談
- 相談タブ下部に国家資格者によるオンライン相談の予告
- ホームにEKIKEN相談へのショートカットを追加
- 予報ページ右上の鉛筆は、従来どおり記録への短縮導線として残す
- AI分析下部のチャットは、選択期間の振り返り専用として残す

## 今の体調相談で毎回使う情報

### 共通知識

- 未病レーダーの体質分類
- 気血水傾向
- 6つの天気ストレス
- 体調ゆらぎ度と3つの予報モード
- 7つのケア方針
- 暮らす・食べる・ほぐす
- 安全上の回答境界

### ユーザー文脈

- 解釈済みの体質トリセツ
- 今日と明日の計算済み予報
- 今日と明日に表示された対策ケア
- 今日に向けて実行済みの具体的ケア
- 直近3日の詳細記録
- 直近14日の集計要約
- 今の体調相談の直近16メッセージ

氏名、メール、住所、体質チェックの生回答は送らない。
OpenAI Responses APIの保存機能は無効化する。

## 会話履歴の分離

`records_ai_threads.thread_kind` を追加する。

```text
period_review
  選択期間のAI分析を深掘りする会話

live_support
  今の体調を相談する会話
```

期間振り返りAPIは `period_review` だけを読み書きし、今の体調相談APIは `live_support` だけを読み書きする。画面上は同じEKIKENでも、履歴とプロンプトを混ぜない。

## 最初の声かけ

完全ランダムではなく、以下から状況に合う候補を選ぶ。

- 今日の予報モード
- 昨日の実感
- 今日の対策ケア実施状況
- 現在の時間帯

症状を決めつける表現や、ケアの効果を断定する表現は使わない。

## クイック相談

初期候補は以下。

- 今、つらい症状がある
- 今日のケアを一緒に選びたい
- なんとなく調子が悪い
- 受診した方がよいか迷っている

候補は入力欄へ入れるのではなく、タップでそのまま送信する。

## 安全設計

EKIKENは以下を行わない。

- 診断、治療、症状原因の特定
- 受診不要の保証
- 薬、漢方、サプリの個別選択・開始・中止・用量変更
- アプリにないケアを「未病レーダーの提案」として創作
- 医師、鍼灸師、登録販売者など人間の資格者を装う

突然の激しい症状などは外部AIへ送らず、固定の安全案内を返す。
強い症状、長引く症状、薬や治療判断は、医療機関または国家資格者への相談を案内する。

## 利用回数

今の体調相談は、既存の期間振り返りチャットと同じ `chat_response` 枠を使用する。
無料・先行体験中の回数制限は両方のチャットで共有する。

## DB migration

コードのデプロイより先に実行する。

```text
supabase/migrations/20260714_add_ekiken_live_support_v7720.sql
```

確認SQL:

```text
supabase/checks/20260714_check_ekiken_live_support_v7720.sql
```

ロールバックSQLはlive_support会話を削除してから列を戻すため、実行時は会話データ消失に注意する。

## 環境変数

専用モデルを分ける場合だけ設定する。

```text
OPENAI_RECORDS_LIVE_CHAT_MODEL=gpt-5.6-luna
```

未設定時は `OPENAI_RECORDS_CHAT_MODEL`、さらに未設定ならコード既定値を使う。

## 現時点の制約

- 毎回AIへ渡す会話履歴は直近16メッセージ。`context_summary` 列は将来の長期会話要約用に準備しているが、v7.72.0では自動要約をまだ生成しない
- 質問に応じて30〜90日分の関連記録を追加検索する検索エージェントは未実装。現在は直近3日詳細＋14日要約
- EKIKENが会話中に提案したケアを、ボタン一つで具体的ケア記録へ保存する機能は未実装。予報ページの対策ケア記録とは出自設計を分けてから追加する
- 国家資格者によるオンライン相談は予告段階
