# v7.72.5 Ekiken確認質問・回答コンテキスト保持

## 目的

Ekikenの確認質問は、チャット本文とは別のカードとして表示される。v7.72.4までは、回答候補を選んだ後に質問カードが消え、保存されるユーザー発言は回答文だけだった。

そのため、AIは直前の本文から回答先を推測できることが多かったものの、短い回答や後日の履歴確認では、何への返答か分からなくなる可能性があった。

v7.72.5では、確認質問と回答を一組として保存・表示・AI入力へ渡す。

## 実装

- 回答候補を押した時に、次を入力欄の回答コンテキストとして保持する
  - 確認質問を出したassistant message ID
  - 質問文
  - 選択した候補
  - follow-up kind / date
- 候補を編集して送っても、元の質問との対応を維持する
- 候補を使わず自由入力で答えた場合も、確認カードが有効な間は質問への回答として保持する
- 別の話題へ移る場合は「この質問には答えず、別のことを話す」で紐づけを解除できる
- サーバー側で、指定されたassistant messageと保存済みfollow_upを照合し、ユーザーが任意の質問文を偽装できないようにする
- 検証済みの対応関係を`records_ai_messages.metadata.reply_to_follow_up`へ保存する
- 会話履歴では、ユーザー回答の吹き出し内に元の質問を小さく表示する
- AIへ渡す直近会話にも`reply_to_follow_up`と自然文脈を含め、推測ではなく質問への回答として解釈させる

## 対象

- 今の体調を相談するEkikenチャット
- 選択期間を振り返るAIチャット

## 互換性

既存メッセージは`reply_to_follow_up`を持たないため、従来どおり本文だけを表示する。既存データの変換は行わない。

## DB

既存の`records_ai_messages.metadata` JSONを利用するため、追加migrationは不要。
