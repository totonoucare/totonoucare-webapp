# 注意時間の変化方向と水分環境UI v7.78.20

## 注意時間の意味

注意時間は、症状が必ず始まる時刻ではない。各気象負荷について、予測値が最も強くなる連続3時間前後を示す。

絶対環境と変化では意味が異なる。

| 表示 | 注意時間 |
|---|---|
| 高温 | 高温負荷が最も強い時間帯 |
| 低温 | 低温負荷が最も強い時間帯 |
| 気温上昇 | 上昇量・上昇速度による負荷が最も強い時間帯 |
| 気温低下 | 低下量・低下速度による負荷が最も強い時間帯 |
| 気圧上昇 | 気圧上昇の変化負荷が最も強い時間帯 |
| 気圧低下 | 気圧低下の変化負荷が最も強い時間帯 |
| 湿気 | 湿気側の絶対環境または増湿負荷が最も強い時間帯 |
| 乾燥 | 乾燥側の絶対環境または水分低下負荷が最も強い時間帯 |

## 混合日の表示

一日の中で上昇・低下がともに強い場合、カードの気象アイコンは一日全体の特徴を表す。

- 気温: `寒暖差`
- 気圧: `気圧変動`

注意時間は、そのうち負荷が強い方のピークを一つ表示する。

- 暖色の上矢印: 上昇側ピーク
- 青色の下矢印: 低下側ピーク
- 旧データなどで優勢方向を決められない場合: 紫色の上下矢印

これにより、カードでは一日の混合性を残しつつ、ユーザーが実際に備える時間の方向を読める。

## 水分環境を2表示に保つ理由

水分環境は`湿気 / 乾燥`の2表示を維持し、`湿度差`を第3分類にしない。

- 主に絶対湿度を用いるため、相対湿度の数値変化と混同させない
- 増湿は湿気側、乾燥方向は乾燥側としてケアへ直接つなげやすい
- 変化と絶対環境を別カードにすると、同じ水分現象を二重表示しやすい
- 体調ゆらぎ度では`moisture_shift`をすでに急性変化として評価している

湿度カードでは、絶対環境負荷と変化負荷の強い方を採用する。

ただし、中間帯へ戻る変化を反対側の環境とは呼ばない。

- 湿った環境から中間帯へ戻る水分低下: `湿気`側の負荷が弱まりつつある変化
- 乾いた環境から中間帯へ戻る水分上昇: `乾燥`側の負荷が弱まりつつある変化
- 中間帯から湿った側へ離れる: `湿気`
- 中間帯から乾いた側へ離れる: `乾燥`

## データ契約

方向別タイムラインとピークへ、次を追加する。

- `pressure_shift / pressure_up / pressure_down`
- `temp_shift / temp_up / temp_down`
- `moisture_shift / moisture_up / moisture_down`
- 各ピークの`strength`
- 表示中の時間方向を示す`attention_direction`

既存の`direction`は一日全体の物理方向を表し、`attention_direction`は表示中のピーク時間の方向を表す。両者を混同しない。
