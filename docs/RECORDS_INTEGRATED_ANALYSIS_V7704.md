# v7.70.4 記録・分析 改良版

## 目的

予報ロジックは記録によって変えず、`ベース体質 × 天気ストレス`から算出した体調ゆらぎ度を固定された比較条件として扱う。
記録は、似た予報条件におけるケアの有無・内容・タイミングと実感の関係を振り返るために使う。

## 分析画面

### 統合時系列グラフ

1つの時系列に次を配置した。

- 上段：体調ゆらぎ度 0〜100
- 背景：安定／いたわり／守りの範囲
- 日別ラベル：主な天気ストレス
- 中段：実感 ○／△／×（体調ゆらぎ度とは別尺度）
- 下段：暮らす／食べる／ほぐすとケア時刻

3か月以上では週単位にまとめ、体調ゆらぎ度の平均・最小・最大、実感の内訳、ケア日数を表示する。

### 4つの振り返りパターン

大型カードを廃止し、グラフ内の絞り込みとして統合した。

- 注意予報でも穏やか
- 注意予報でつらさあり
- 安定予報どおり穏やか
- 安定予報でもつらさあり

絞り込みは予報の当たり外れを評価するものではなく、比較したい日を探すために使う。

## AI分析

サーバー側で、次の条件が近い日をまとめた `matched_forecast_comparisons` を作る。

- 主な天気ストレスが同じ
- 体調ゆらぎ度が同じ15点幅の帯

各グループで、ケアあり／なし、実感○△×、ケア分野、注意時間前／つらくなってからを集計する。
AIには計算済みの比較を優先して解釈させ、条件の異なる日を単純比較させない。

## 予報への記録フィードバック

記録による天気親和性補正は標準で無効にした。

```env
RADAR_REVIEW_FEEDBACK_ENABLED=false
```

検証環境で明示的に `true` を設定した場合だけ、既存の補正経路を有効化できる。
