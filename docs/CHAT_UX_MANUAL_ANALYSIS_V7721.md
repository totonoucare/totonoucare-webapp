# v7.72.1 チャットUX・AI分析手動生成

## 目的

EKIKENを「候補を押すだけで勝手に送信される画面」や「毎回受診確認を重ねる画面」にせず、本人が言葉を調整できる聞き上手な相談役へ近づける。AI分析は、タブを開いたこと自体では回数を消費しない。

## クイック候補

- 今の体調相談の一言候補
- EKIKENからの確認選択肢
- 期間振り返りチャットの確認選択肢

上記はいずれも入力欄へ転記する。ユーザーが書き足し・修正した後に送信する。

## 自動スクロール

今の体調相談と期間振り返りチャットの両方で、次の場合に会話末尾へ追従する。

- 履歴読み込み完了
- ユーザー送信直後
- 応答待ち表示
- EKIKENの返信受信

## 受診・相談状況

相談画面で任意登録する。選択肢は次のとおり。

- この不調はまだ相談していない
- 医療機関に相談・検査中
- 検査では大きな異常なしと言われた
- 医療機関で治療中
- 鍼灸院・治療院などに相談中
- 今は登録しない

この操作はチャットメッセージではなく、回数を消費しない。既存の `records_ai_threads.context_summary` に保存するため追加migrationは不要。会話を削除しても、この状況だけは保持する。

症状ごとに状況が異なる可能性があるため、AIは絶対的な事実として断定せず、同じ質問を繰り返さないための参考情報として扱う。

## 安全設計の調整

- 明確な緊急サインは従来どおり外部AIへ送る前に固定判定する
- 通常の体調相談を受診判定の問診から始めない
- 受診状況が登録済みなら、毎回同じ質問を繰り返さない
- ユーザーが受診判断を相談目的として選んだ時、または新しい急激・強い変化がある時だけ確認を優先する
- 緊急でない専門領域の話は会話を止めず、一般的セルフケアの整理を続け、必要な相談先を短く添える

## AI分析

`POST /api/records/analysis` に `generate` を追加する。

### generate: false

- 現在の記録と完全一致する保存済み分析があれば表示
- 現在の記録とは一致しないが同じ期間の保存済み分析があれば、古い内容であることを示して表示
- 保存済み分析がなければ基本集計による分析だけ表示
- OpenAI APIを呼ばず、分析回数を消費しない

### generate: true

- 現在の記録と一致する保存済み分析があれば再利用
- 一致しなければ上限確認後に新規生成

画面では、未生成時は `EKIKENに聞く`、記録更新後は `現在の記録で分析を更新` を表示する。

## 同意

任意登録した受診・相談状況をAIへ送る可能性があるため、同意バージョンを次へ更新する。

```text
records_ai_v5_consultation_context_2026-07-14
```

```text
体質・予報・ケアをつなぐ
```
