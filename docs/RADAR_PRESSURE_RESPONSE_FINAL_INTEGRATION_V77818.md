# 気圧反応の互換性とEkken接続仕上げ v7.78.18

## 目的

v7.78.17で導入した「気圧の物理方向」と「体質による身体反応」の分離を、旧保存データ、Ekken、体質トリセツまで一貫させる。予報スコアや体質親和性の係数は変更しない。

## 旧データ互換

反応方向が保存されていない旧予報と、明示的な`balanced`を区別する。

| 保存状態 | ケアへの投影 |
|---|---|
| 反応方向なし＋`pressure_down` | 旧`pressure_down`ケアを維持 |
| 反応方向なし＋`pressure_up` | 旧`pressure_up`ケアを維持 |
| 明示的な`accel` | 張り・力み側の互換キーへ投影 |
| 明示的な`brake` | 重だるさ側の互換キーへ投影 |
| 明示的な`balanced` | 適応疲れ用の中立ケアへ投影 |

欠損を自動的に`balanced`とみなすと、V1や保存済み当日予報のケアが変わるため禁止する。

## Ekkenへ渡す契約

予報の上位と各気象factorへ、次を明示的に渡す。

- `personal_main_event_key`
- `pressure_direction`
- `pressure_response_direction`
- `physical_direction`
- `response_direction`
- `body_response_key`
- `environmental_cautions`

Ekkenは「気圧は低下、身体反応はアクセル側」のように物理と反応を別々の事実として受け取り、入れ子の体質情報から推測し直さない。

## 体質トリセツ

天気相性ランキングは従来どおり、本人回答、寒熱回答、環境感受性、気血水6傾向を含むV2親和性で決める。気圧カードの名称は上昇／低下を維持するが、本文の張り／重だるさは`yin_yang_score`から求めた反応方向で説明する。

通常の温度名は次の役割に統一する。

- 変化: `気温低下 / 気温上昇`
- 絶対環境: `低温 / 高温`
- 極端域の別枠注意: `厳しい寒さ / 猛暑 / 酷暑`

## 結合テスト

`tests/radar-pressure-response-integration-v77818.test.mjs`で、次の4組を実際の関数出力として検証する。

| 物理方向 | 身体反応 | 本文・ケア |
|---|---|---|
| 低下 | アクセル | 張り・力み |
| 低下 | ブレーキ | 重だるさ |
| 上昇 | アクセル | 張り・力み |
| 上昇 | ブレーキ | 重だるさ |

検証範囲は、天気カード、不調サイン、この日の方針、暮らす・食べる・ほぐす、ツボ説明、ショップ選定キー、Ekken JSON、体質トリセツ。

## 維持するもの

- V2の気象抽出、快適域方向校正、体質親和性、余力補正
- 体調ゆらぎ度の計算式と閾値
- 今日01:05固定、明日16:30再計算
- 未ログイン／ログインの共通V2エンジン
- `RADAR_FORECAST_MODEL_VERSION=v1`によるロールバック
- 東洋医学の状態を理解しやすくする、意味の通る比喩表現
