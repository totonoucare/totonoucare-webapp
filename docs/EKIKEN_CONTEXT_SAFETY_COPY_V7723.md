# v7.72.3 文脈を考慮した安全分岐・同意文統一

## 目的

v7.72.2で増やした緊急表現の言い換え対応を維持しながら、否定・過去・引用・第三者相談まで固定安全ルートへ入る過剰反応を減らす。通常相談を不必要に止めず、明確な現在の緊急事態だけをアプリ側で固定処理する。

## 安全分岐

`classifySafetyText()` は次を区別する。

- `current_direct`: 本人の現在の直接的な訴え。固定安全ルートへ入る
- `negated`: 「胸が痛いわけではない」などの否定
- `past`: 「昔、胸が痛い時期があった」など明確な過去
- `third_party_or_quote`: 引用や家族・友人など第三者についての相談

後者3つは通常会話を止めず、`potential_safety_signal` としてモデルへ渡す。モデルは文脈に沿って応答し、この情報だけで本人の現在の緊急状態と断定しない。

## 同意文

AI利用前カードと使い方ガイドを次の意味へ統一した。

- アカウントに登録された氏名・メールアドレス・住所は自動送信しない
- 体質チェックの生回答は自動送信しない
- 記録メモや会話欄へ本人が入力した内容は送信対象になる

AIへ送るデータ範囲自体はv7.72.2から変更していないため、同意バージョンは変更しない。

## 会話の長さ

Ekikenの通常回答は300〜500文字程度を目安にする。安全案内や重要な説明に必要な場合のみ、これより長くしてよい。
