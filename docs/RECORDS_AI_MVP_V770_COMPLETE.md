## ユーザー導線

1. ホームまたは体調予報から「今日を振り返る」へ進む。
2. 体調、先回りケア、ケアの種類・時刻、任意メモを記録する。
3. 予報と実感がずれた日だけ、生活側で思い当たる条件を任意記録する。
4. カレンダーで当日の予報・実感・ケアを見返す。
5. AI分析タブで期間集計、実感中心の推移、4つの振り返りパターンを見る。
6. 明示同意後、同じ期間の記録を引き継いだAI分析・チャットを使う。
7. 人に相談したいユーザーは専門家相談の「利用してみたい」を送る。

## `/records` の3タブ

### 記録カレンダー

- 体調: `○ よかった / △ 少しつらかった / × つらかった`
- 先回りケア: `した / 少しした / していない`
- ケアをした場合: `暮らす / 食べる / ほぐす` を複数選択
- ケア時刻: `注意時間の前 / つらくなってから / 覚えていない`
- 予報と実感がずれた場合だけ: `寝不足 / 忙しさ / 食事・飲酒 / 気分の負担 / 生理・体調変化 / その他 / 特にない`
- 任意メモ: 最大500文字
- 日本時間で1日1件。同じ日は編集扱い。
- 今日を含む直近7日まで追加入力・編集可能。それより前は閲覧のみ。
- 最初に保存した予報スナップショットはDBトリガーで固定し、後日の予報再計算で過去比較が変わらないようにした。

生活条件は予報アルゴリズムへ戻さない。あくまで「予報と実感の差を振り返る材料」として保存する。

### AI分析・相談

- 期間: 1週間、1ヶ月、3ヶ月、6ヶ月、12ヶ月
- 1週間・1ヶ月は日単位、3ヶ月以上はISO週単位で集約
- 表示: 記録日数、穏やかな日、つらさがあった日、先回りケアをした日
- 時系列: 実感の○△×を主役にし、背景へ予報モード、下段へケア種別を表示
- 4パターン: 注意予報／安定予報 × 穏やか／つらさあり。日数、ケア有無、日付詳細を表示
- 3ヶ月以上: 週平均値を折れ線にせず、週内の○△×件数を表示
- 同じ天気・同じケアが2回以上ある時だけ反復パターンとして表示
- AI文章: 受容 → 確認できる事実 → 仮説 → 次の小さな試し方 → 追加質問
- AI会話: 未病レーダー共通知識、解釈済み体質、計算済み予報根拠、表示ケア、集計値、必要な日次記録、直近会話をサーバー側で渡す
- AI回答評価: 👍 / 👎 と理由
- 会話削除: ユーザー操作でスレッドとメッセージをDBから削除

## 予報と実感の振り返りロジック

予報モードは症状の予測値ではなく備えの目安であるため、的中／外れではなく4パターンへ整理する。

| 予報＼実感 | ○ よかった | △・× つらさあり |
|---|---:|---:|
| 安定 | 安定予報どおり穏やか | 安定予報でもつらさあり |
| いたわり・守り | 注意予報でも穏やか | 注意予報でつらさあり |

予報が存在しない日は「安定」と見なさず、`予報なしの記録` として比較対象から外す。

## ロジックとAIの役割分担

### アプリ側で確定する事実

- 予報・実感の4パターン集計
- ケアあり／なし、種類、時刻
- 注意／安定予報と、穏やか／つらさありの組み合わせ日数
- 気象トリガー別・ケア別の反復回数
- 長期表示の週内○△×件数
- AI利用回数、同意、権限、キャッシュ判定

### AIが担当すること

- 複数の記録・体質・天気を自然な言葉で整理
- 一つの原因に決めつけず、複数の仮説を区別
- 予報との差をごまかさず、次に確認する条件を提案
- 本人が答えやすい追加質問を一つずつ提示
- 寄り添いながらも依存を促さない会話

AIには件数計算や効果判定を任せない。「ケアが効いた／効かなかった」は断定せず、同条件で再現するかを見る。

## AI安全・プライバシー

- 外部AI送信前に、送信項目を明記した同意をDBへ保存
- 氏名、メール、住所はプロンプトへ入れない
- 体質チェックの生回答は送らず、人間向けに解釈した体質要約だけを送る
- 内部ユーザーIDはHMACで不可逆化した `safety_identifier` に変換
- OpenAI Responses APIは `store: false`
- 会話履歴の正本はSupabase側。送信する履歴は直近に制限
- 記録メモ内の命令文をプロンプト指示として扱わない
- JSON Schemaで安全レベル・追加質問を必須化
- 診断、治療断定、薬・漢方・サプリの個別な使用・中止・増減判断は禁止
- 緊急語は外部AI呼び出し前にルール検知し、利用上限中でも救急・緊急相談を優先案内
- 薬・漢方・サプリ、受診・治療判断の質問はルールでも検知し、専門家確認を必ず付ける
- AIへの送信同意は取り消し可能。既存のAI会話は別操作で削除可能
