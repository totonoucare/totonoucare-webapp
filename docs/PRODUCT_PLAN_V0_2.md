# 未病レーダー 開発計画 v0.2 要約

最終更新: 2026-05-08

## コアメッセージ

> 明日の崩れやすさを、今夜の整え方に変える。

未病レーダーは、単なる「今日の不調を当てるアプリ」ではなく、明日の崩れやすさを先回りし、今夜から整えるための未病予報アプリとして設計する。

---

## 画面ごとの役割

### ダッシュボード

役割:

- 今日ここからの軽いリアルタイム予報
- キャラクターによる一言コメント
- 明日の未病予報への導線
- パーソナル体質トリセツへの導線

今日予報は:

- AI生成なし
- DB保存なし
- MET Norwayから取得できる「今以降〜今日終わりまで」を使ってルールベース表示
- 医療的対処ではなく「このあと重くなりやすい時間」「今できる軽い一手」を見せる

### 予報ページ

役割:

- 明日の未病予報をしっかり見る本番ページ

構成:

```text
明日の未病予報
↓
アニメーションスコアゲージ
↓
響きやすい要素 1〜2個
↓
明日の山場
↓
AI生成の読み解き文
↓
今夜の先回りケア
  ほぐす / 食べる / 暮らす
```

「明日の山場」は時間帯を端的に伝える。  
余計なルール作文は入れず、詳細な注意点はAI生成文に任せる。

## ケア提案UI

カードを縦に増やすのではなく、以下のタブに分ける。

```text
ほぐす / 食べる / 暮らす
```

### ほぐす

既存のツボケア。  
内容:

- 選ばれたツボ
- どんなとき向きか
- 押し方
- 選定理由

### 食べる

既存の食養生カード。  
今後は「当たり前の健康アドバイス」から「身体の読み解き」に進化させる。

弱い例:

> 湿気の日は脂っこいものや甘いものを控えましょう。

目指す例:

> この日は、食べたものが力になる前に「重さ」として残りやすい日。冷たい麺と甘いカフェラテを重ねるより、温かい汁物に香味野菜を少し足す方が、流れを作りやすい。

### 暮らす

新設タブ。  
身の回りでできる東洋医学的ライフハックを担当する。

例:

- 湿気の日の寝室や洗濯物の置き方
- 気圧低下の日の耳まわり、首肩、照明、音の扱い
- 冷え込みの日の足元、腹腰、寝具
- 乾燥の日の寝室、喉、湯気、加湿
- 暑湿の日の風の通り道、寝苦しさ対策

「無料でできる」のような開発者メモ的文言は避ける。  
使うなら:

```text
今夜の支度
暮らしの一手
身の回りケア
```

## アフィリエイト・物販導線

採用方針:

- 「バーチャル養生キットカードを1枚追加」は採用しない。
- モバイルで縦長になりすぎるため。
- 広告カード感が出やすいため。
- 代わりに `ほぐす / 食べる / 暮らす` 各タブ内に自然にPR導線を差し込む。

最終形:

```text
人間が商品カタログを作る
↓
商品やカテゴリにタグを付ける
↓
体質・天気・時間帯とスコアリングで候補を選ぶ
↓
AIは候補内で見せ方や説明を整える
```

商品選定は if文分岐地獄にしない。  
タグ重み × 予報・体質・時間帯のスコアリングで、小さな推薦エンジンとして設計する。

PR表示方針:

- PR表示は必須。
- 広告枠っぽく見せない。
- 避ける表現: おすすめ商品 / 今すぐ買う / 効く / 改善する / 治る / 痛みを和らげる
- 使いたい表現: この日の支度 / 今夜の備え / 暮らし方に合わせたアイテム / PRを含みます

## 実装ステップ

### Phase 1: 明日予報UXの仕上げ

- ダッシュボード/予報ページの文言調整
- 響きやすい要素2チップ表示確認
- 暮らすタブのラベル調整
- 今日ここからカードの見た目調整

### Phase 2: ケア提案の質向上

- 食べるタブAIプロンプト強化
- 暮らすタブのライフハック拡充
- ほぐすタブの見せ方微調整

### Phase 3: アフィリ導線の器作り

- 各タブ下部にPR枠を差し込める構造を作る
- feature flagで制御
- 最初は商品リンクなし/仮リンクでも可

### Phase 4: 商品カタログと推薦エンジン

- 商品カテゴリ定義
- 商品カタログ定義
- タグ重みスコアリング
- 体質×天気×時間帯で候補選定

### Phase 5: クリック計測とD2C検証

- クリックイベント保存
- 体質/天気/カテゴリ別CTR分析
- 反応の良いカテゴリを養生キット候補化
- 将来の自社D2C/OEMへ接続

## 意思決定ログ

### 1. 予報ページは「明日の未病予報」を主役にする

当初は今日/明日のタブ切り替えで予報を見せていたが、未病レーダーの価値は「今しんどい症状を当てること」ではなく、「明日の崩れやすさを先回りして今夜の整え方に変えること」にあると整理した。

今日の不調に対する即時対処は、頭痛ーるや一般的な頭痛薬・休息などの領域と競合しやすく、東洋医学的セルフケアの価値が伝わりにくい。

そのため、予報ページは明日の未病予報に集中させ、今日ここからの軽い確認はダッシュボードに分ける。

### 2. 今日予報はダッシュボードで軽く扱う

今日予報は、AI生成やDB保存を行わず、ページを開いた時点から今日の終わりまでの未来予報だけを使ってルールベースで表示する。

目的は「今日の不調を診断すること」ではなく、「このあと少し気をつけたい時間」と「今できる軽い一手」を伝えること。

### 3. 予報ページの主役はアニメーションスコアゲージ

明日予報に寄せた後も、予報ページの主役は既存のアニメーションスコアゲージとする。

ゲージで明日の崩れやすさを印象づけ、その下に響きやすい要素、明日の山場、AI生成の読み解き、ケアタブを続ける。

### 4. ケア提案は「ほぐす / 食べる / 暮らす」に分ける

ツボケア、食養生、物販/アフィリ導線を縦に並べると、モバイルでは長くなりすぎる。

そのため、予報ページのケア提案は「ほぐす / 食べる / 暮らす」の3タブに分ける。

- ほぐす: ツボ、経絡、セルフケア、温熱、ケアグッズ
- 食べる: 食養生、飲み物、食材、養生茶、食品系
- 暮らす: 部屋、睡眠、入浴、香り、湿度、服装、身の回りの工夫

### 5. バーチャル養生キット単独カードは採用しない

当初は「バーチャル養生キット」カードを追加する案もあったが、予報ページが縦長になり、広告カード感が強くなる懸念があった。

そのため、商品導線は単独カードではなく、各ケアタブの文脈内に自然に差し込む方針とした。

### 6. アフィリは広告枠ではなく「行動を実装する道具」として置く

アフィリ導線は、単に商品を売るための枠ではなく、体質×天気で提案されたケアを実行しやすくするための道具として扱う。

ただし、コピーは弱くしすぎない。  
効能断定は避けつつ、身体の読み解き、比喩、作戦名で「知らない角度からの納得感」を作る。

### 7. AIに商品をネットから勝手に拾わせない

最終形でも、AIがネットから商品を自動収集する設計にはしない。

人間が事前に商品カタログを作り、商品やカテゴリにタグを付ける。  
その上で、体質・天気・時間帯とのスコアリングで候補を絞り、AIは候補内で説明文や見せ方を整える。

### 8. 商品選定はif文分岐ではなくスコアリングにする

「湿気×冷えならA」のような分岐地獄にはしない。

商品カテゴリや商品に weatherWeights / constitutionWeights / sceneWeights / roles を持たせ、予報・体質・時間帯との一致度で選ぶ小さな推薦エンジンとして設計する。
