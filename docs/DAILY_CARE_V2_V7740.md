# v7.74.0 Daily Care v2

## 目的

従来のDaily Careは、天気条件から完成済みの文章セットを選ぶ部分が多く、同じような予報が続くと内容が固定化しやすかった。食べるカードも、食材候補・飲み物・注意・理由を最初から多く表示するため、定番的なのに冗長と感じやすかった。

v7.74.0では、予報ロジックを変えず、ルールが決める対象を「完成文章」から「今日のケア方向・刺激量・候補の適合度」へ移した。

## 共通ケアテーマ

`lib/radar_v1/careRules/dailyCareV2.js` が、次の材料から共通テーマを作る。

- 計算済みの主因・副因
- 安定／いたわり／守りに対応するsignal
- コアタイプと余力
- 気虚・気滞・血虚・血瘀・痰湿・津液不足
- 不調フォーカス
- 主・副経絡ライン

テーマの語彙は次の7方向。

- しずめる
- ゆるめる
- めぐらせる
- ながす
- うるおす
- ぬくめる
- ささえる

このテーマを、暮らす・食べる・ほぐすの3カテゴリで共有する。各カテゴリは同じ言葉を繰り返すのではなく、それぞれの行動へ翻訳する。

## 候補のローテーション

候補は完全な乱数で選ばない。対象日とケア文脈から安定したindexを作る。

- 同じ日・同じ条件・再読み込みでは同じ候補
- 日付が変わると、適合度が近い候補の中で主提案が入れ替わる
- 体質や予報に合わない候補を、変化のためだけに出さない

この仕組みにより、予報結果の再現性を保ちながら、連日同じ文章になる問題を抑える。

## 暮らす

主表示は「今日の一手」1件と短い理由。別案は最大2件で、注意点とともに折りたたみ内へ置く。

安定／いたわり／守りと余力に応じて、行動のタイミングと刺激量を変える。守りや余力小では、強い運動や長時間の介入ではなく、短く弱い行動を優先する。

## 食べる

主表示は、食材一覧ではなく具体的な一食1件。

例:

- 温かいそば＋大根おろし・しそを少し
- おにぎり＋茶碗蒸し＋温かいお茶
- 蒸し鶏と夏野菜＋少量のごはん

別案は最大2件。飲み物、控えたい組み合わせ、詳しい理由は必要な人だけ開く。従来の詳細な食材候補は `ingredient_suggestions` に残し、体質ラベルによる選定互換性を維持する。

## ほぐす

登録済みの主経絡ラインを「今日の一手」として優先し、体のラインをさする・温める・軽く動かすケアを出す。ツボは補助候補として表示する。

今日のツボ選定は次を点数化する。

- 不調フォーカス
- 主因・補助トリガー
- 主経絡・副経絡
- 気血津液の上位傾向
- 余力小
- 対象日内で安定する小さな順位差

経絡ラインケアは `tsubo_line_care` として実行記録できる。

## 互換性

- 体調予報の点数、モード、主因、副因、親和性、有効負担は変更しない
- 既存のcare plan JSONを読み込んだ場合も、画面表示時にDaily Care v2へ変換できる
- 既存の食材選定結果は `ingredient_suggestions` に保持する
- 既存の生活ケア・食事・ツボ記録とのcanonical identity互換性を維持する
- DB migration、環境変数変更、AI同意更新は不要
