# v7.78.23 天気負荷ピークの意味統合

## 目的

`peak_start / peak_end`が示すのは気象負荷のピークであり、身体症状が発生・悪化する時刻ではない。この契約を予報画面だけでなく、生成文、ケア、記録、AI相談まで一貫させる。

## 表示語彙

- 気象の時刻: `天気負荷が強まる時間帯`
- ピークピル: `天気負荷のピーク`
- ケア時刻: `天気負荷のピーク前`
- 症状が起きた後: `つらくなってから`

`注意時間帯`、`崩れやすい時間帯`、`響きやすい時間帯`は使わない。

## AI境界

予報生成、記録分析、期間チャット、ライブ相談へ次を明示する。

- `peak_start / peak_end`は気象現象のピーク
- 症状の発生・悪化時刻ではない
- この時間に症状が出ると断定しない
- 体質は症状の表れ方を決めるが、症状時刻を確定しない

プロンプト内容が変わるため、records系3経路のバージョンを更新する。

## 旧保存データ

v7.78.23では旧保存文を表示時に正規化していたが、予報GPTと旧文互換はv7.78.25で削除した。
現在の`why_short`はルール生成時点で以下の正本語彙を保存する。

- `注意時間`
- `注意時間帯`
- `崩れやすい時間帯`
- `響きやすい時間帯`

## 深夜表記

終了時刻が開始時刻より小さい場合は日付をまたぐものとして扱う。

- 変更前: `21–0時`
- 変更後: `21時–翌0時`

## 維持する内部契約

- `peak_start`
- `peak_end`
- `before_peak`
- ピークの計算方法と代表選択
- 体調ゆらぎ度
- 体質親和性・余力
- ケア選定条件
- V1ロールバック
- DB、SQL、環境変数
