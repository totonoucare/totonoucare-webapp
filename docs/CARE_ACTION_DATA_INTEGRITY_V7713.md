# v7.71.3 ケア記録の出自・識別・表示調整

## 方針

有料分析の中心となる「アプリが提案した具体的ケア」と、本人がアプリ提案外で行った行動を混同しない。
先行公開では、具体的ケアの入力入口を予報ページのDaily Careカードへ一本化する。

## 変更

### Daily Care由来だけを再現性分析へ使用

- `performed_care_items`: Daily Careで提案され、本人が「やってみた」と記録したケア
- `user_added_care_items`: 本人が別経路で追加したケア。互換用に入力構造は残すが、具体的ケアの再現性集計には混ぜない
- 記録ページの自由入力UIは先行公開では非表示

### ケアID v3

- ツボ: ツボコードを優先
- 食べる: カード種別と食品名を中心に識別
- 暮らす: 固定ルールIDがある場合は優先し、ない場合はケア本文を主IDにする
- 説明文や表示順の変更では別ケアにならない

### 旧重複行の削除

削除対象の行からcanonical keyと表示元を復元し、同じ意味の旧行をまとめて削除する。

### UI

- いたわり: 眉なしで、少し疲れて汗が出ている表情
- 守り: 眉なしで、目をぎゅっと閉じたつらそうな表情
- AI画面の英語見出しと開発都合語を自然な日本語へ置換
