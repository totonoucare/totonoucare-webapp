# 天気アイコンの状態・方向分離 v7.78.19

## 目的

予報V2では、気温の「変化」と「絶対的な高温・低温」を別々に計算している。一方、表示層では旧来の`cold / heat`キーと6種類のSVGを流用していたため、次の不整合があった。

- `temp_shift`を直接受け取る未ログインデモと体質トリセツで、アイコンが表示されない
- 気温低下と低温、気温上昇と高温が、同じ意味として扱われる
- `mixed`の寒暖差が、方向情報に関係なく気温上昇アイコンへ寄る
- 気圧の上下変動が、優勢方向だけのアイコンになる

この版では計算ロジックを変えず、表示契約だけを整理する。

## 表示契約

| 計算イベント | 物理方向 | UI表示 | アイコン |
|---|---|---|---|
| `pressure_shift` | `down` | 気圧低下 | 既存・外向きの気圧低下 |
| `pressure_shift` | `up` | 気圧上昇 | 既存・内向きの気圧上昇 |
| `pressure_shift` | `mixed` | 気圧変動 | 新規・上下する2ベクトル |
| `temperature_shift` | `up` | 気温上昇 | 既存・上昇する熱気 |
| `temperature_shift` | `down` | 気温低下 | 新規・下降する冷気 |
| `temperature_shift` | `mixed / change` | 寒暖差 | 新規・暖気上昇と冷気下降 |
| `heat` | 状態 | 高温 | 新規・強い太陽と日照 |
| `cold` | 状態 | 低温 | 既存・氷結 |
| `damp` | 状態 | 湿気 | 既存・水滴と重い空気 |
| `dry` | 状態 | 乾燥 | 既存・蒸発とひび割れ |

## 用語

ユーザー向けの総称は`寒暖差`へ統一する。

- 上向きだけが目立つ変化: `気温上昇`
- 下向きだけが目立つ変化: `気温低下`
- 上下が重なる、または向きを限定しない変化: `寒暖差`
- 内部の計算概念: `temperature_shift`

旧保存データに含まれる`気温差が大きい日`は、通知生成時の互換入力としてのみ受け付け、表示時に`寒暖差`へ正規化する。

## 変更しないもの

- 体調ゆらぎ度の計算式と閾値
- 気象6イベントの強度
- 体質親和性と余力補正
- 主因・副因の順位
- ケア、ツボ、ショップ、Ekkenの選定
- 今日01:05固定、明日16:30再計算
- `RADAR_FORECAST_MODEL_VERSION=v1`へのロールバック
- DBと保存形式
