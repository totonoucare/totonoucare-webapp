# v7.72.9 Ekken 体質チェック階層理解の強化

## 目的

Ekkenへ渡す体質情報を、単なるプロフィールタグの集合から、体質チェック実装に沿った階層構造へ変更する。

これまでのAIコンテキストには、チーター型、気虚、気滞、主・副経絡などの結果は含まれていたが、次の関係が十分に明示されていなかった。

- コアタイプは最上位の統合結果
- アクセル／ブレーキ軸と余力軸がコアタイプを構成する
- 気滞・気虚・血虚・血瘀・痰湿・津液不足は、その内訳を説明する下位要素
- 不調フォーカスと主・副経絡は、現在どこへどう現れやすいかを見る材料

そのため、Ekkenが漢方相談などで「気虚＋気滞」という一般的な組み合わせだけを拾い、コアタイプを十分に使わない問題があった。

## 実装上の体質モデル

### アクセル／ブレーキ軸

性格や活動量ではなく、不調時に表へ出やすい反応方向を示す。

- アクセル側: 主に気滞
- ブレーキ側: 主に痰湿に、気虚と動き始めの重さを補助的に加える
- 主観的な寒熱: 境界付近の補助だけに使用

### 余力軸

単なる気虚ではなく、回復に使えるバッテリーを示す。

- 気虚・血虚・津液不足の量的負荷
- 消耗を翌日へ持ち越しやすいか
- 環境感受性

をまとめて、小・標準・大へ分類する。

### 滞り補助軸

気滞・血瘀・痰湿をまとめた0〜1の内部補助軸。コアタイプを決める第三分類軸ではなく、滞り・重さ・固定化を読む材料として使う。

### 下位の6パターン

- 気滞
- 気虚
- 血虚
- 血瘀
- 痰湿
- 津液不足

上位2件は表示用の代表ラベルであり、残り4件を捨てる意味ではない。v7.72.9からは、計算済みの6要素すべてを順位付きでEkkenへ渡す。

## Ekkenの推論順序

1. コアタイプを最上位の統合結果として読む
2. アクセル／ブレーキと余力から、ケアの方向と刺激量を決める
3. 6つの物質・流通パターンから、タイプの内訳と崩れ方を読む
4. 不調フォーカス、主・副経絡、天気、現在の訴えへ落とす

例として、アクセル優位×余力小で気滞と気虚が上位なら、巡らせるだけでなく、余力を守り支える方向を同時に入れる。強い発散だけへ寄せない。

## AIへ追加する情報

- 6つの計算済みパターンスコアと順位
- アクセル／ブレーキ、余力、滞り補助軸
- コアタイプごとの統合的な意味
- コアタイプに応じたケアの方向・刺激量・避けたい一方向化
- 日本語の経絡名（肺経・大腸経など）
- 漢方・食養生・暮らす・ほぐすでの体質利用ルール

生の体質チェック回答は送らない。`computed`に保存された計算済み結果だけを、解釈済み体質トリセツとして渡す。

## 漢方相談

症状名だけで定番処方を列挙しない。

- コアタイプ
- 余力
- 6パターンの順位
- 現在の身体所見

から候補を分け、候補になる理由と、どの所見なら優先度が下がるかを説明する。開始・中止・用量・併用可否の最終判断は従来どおり行わない。

## 表現

日本語の相談では、不要な英単語を混ぜない。たとえば `irritability` は「いらだち」「怒りっぽさ」「神経の高ぶり」など、文脈に合う日本語で表現する。

## データベース

追加migrationなし。既存の `constitution_profiles.computed` に保存済みの計算結果を使う。
