# v7.74.1 食べるタブの「足し算＋引き算」

## 目的

Daily Care v2の食べるタブは、具体的な一食を主役にした一方、控えたい食習慣を詳細欄へ押し込んでいた。
また、詳細の開閉名が「なぜこの食材？」のまま残り、実際に含まれる別案・飲み物・注意点と一致していなかった。

v7.74.1では、食養生を次の2本柱で表示する。

1. 今日取り入れたい具体的な一食
2. 今日控えたい食べ方の重なり

## 引き算ロジック

控えたい内容は、一般的な栄養学の禁止リストから固定表示しない。
次の情報を使って候補を採点する。

- 主な天気ストレスと副次的な天気ストレス
- Daily Care共通方針（しずめる・ゆるめる・めぐらせる・ながす・うるおす・ぬくめる・ささえる）
- コアタイプの余力
- 気虚・気滞・血虚・血瘀・痰湿・津液不足
- 不調フォーカス
- 予報モードの強さ

候補は、寒涼・温熱、甘味・辛味、油、乾燥、飲酒、カフェイン、食べる時間や速度などの「重なり」を中心に作る。
例えば湿気の日でも、毎回「冷たい甘い飲み物」だけに固定せず、体質と状態により次のように変わる。

- 冷たい甘い飲み物を食事や間食へ重ねない
- 揚げ物のあとに甘いデザートまで続けない
- サラダや冷たい麺だけで一食を終えない
- 湿気対策の茶や水を一気に大量に流し込まない
- 強い辛味や酒で無理に巡らせようとしない

同じ対象日・同じ条件では表示を固定し、日付が変わると適合度の高い候補内でローテーションする。

## UI

食べるタブの主表示に、次を同じカード階層で並べる。

- 今日の一手
- 今日は控えたい

詳細欄には、主表示と重複しない次の内容だけを置く。

- 別案
- 飲み物
- 選んだ理由
- 必要な場合の食後ケア

開閉名は「ほかの選び方」「別案・飲み物・選んだ理由」に変更する。
明日分は「別案・飲み物・明日の理由」とする。

## 互換性

- 体調予報の点数・モード・主因・副因は変更しない
- DB migrationなし
- 環境変数変更なし
- 既存の食事ケア記録IDの互換性を維持
