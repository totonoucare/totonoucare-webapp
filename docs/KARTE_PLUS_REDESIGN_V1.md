# 未病カルテ Plus 再設計 v1

## 目的

未病カルテ Plus を、無料カルテの長文化ではなく、既存の体質チェック結果と追加問診を統合した「不調ループ見える化レポート」に作り替える。

無料カルテはすでに、体質タイプ・気血水・天気相性・経絡ライン・今日明日の予報導線まで出している。そのためPlusでは、追加チェックによって以下を取得し、無料カルテでは出せない価値を作る。

- 不調が出やすい時間帯
- 悪化しやすい生活条件
- 楽になりやすい方向
- 睡眠・食事・作業姿勢の負荷
- 回復を邪魔する要因
- 続けやすいケアの好み
- 相談時に使える共有情報

## 名称整理

| 位置 | 推奨名 | 備考 |
| --- | --- | --- |
| 無料結果 | 未病カルテ | 既存名を維持 |
| 有料商品 | 未病カルテ Plus | 既存の強い名称を維持 |
| Plusの中身 | 不調ループ見える化レポート | 新しい価値を明確化 |
| 旧称 | 不調マップ | 原則廃止、または補助表現へ格下げ |
| API/内部 | karte_plus / karte_plus_intake | personalKarte から段階的に移行 |

## 役割分担

### 無料カルテ

- 体質のクセを見る
- 気血水の偏りを見る
- 天気との相性を見る
- 負担が出やすい体のラインを見る
- 今日明日の予報へつなぐ

### 未病カルテ Plus

- 繰り返す不調の流れを整理する
- 体質・天気・生活負荷の重なりを見る
- 減らしたい負担と足したいケアを分ける
- 7方針の優先順位を出す
- 7日間の整え方に落とす
- 専門家に相談するときの共有メモを作る

## Plus深掘りチェック v1

`lib/karte_plus/intakeQuestions.js` に定義。

全12問。

1. 不調の継続期間
2. 出やすい時間帯
3. 悪化しやすい条件
4. 楽になりやすい条件
5. 睡眠の崩れ方
6. 食事・胃腸負荷
7. 日常の作業負荷
8. 回復を邪魔しているもの
9. 取り入れやすいケア
10. 今回知りたいこと
11. まず減らせそうな負担
12. 使いやすいまとめ形式

このチェックは無料で実施可能にし、購入前プレビューまで表示する。全文統合レポートを有料にする。

## 追加チェックの設計原則

- 既存の体質判定を上書きしない
- core_code / sub_labels / env_vectors / meridian を再計算しない
- 追加チェックは診断エンジンではなく、カルテ編集エンジンとして扱う
- 医療的な断定ではなく「出やすい流れ」「重なりやすい条件」「整え方の方向」として表現する

## 不調ループ分析 v1

`lib/karte_plus/loopAnalysis.js` に定義。

入力:

- 既存診断結果

出力:

- loopStage: early_loop / forming_loop / fixed_loop
- durationLabel
- loopSignals
- recoverySignals
- rankedDirections
- recoveryBlocks
- carePreferences
- previewText

- ゆるめる
- めぐらせる
- ながす
- うるおす
- ぬくめる
- ささえる

# 未病カルテ Plus v3 方針メモ

## 商品定義

未病カルテ Plus は、無料カルテの延長ではなく「不調ループ見える化レポート」として扱う。

無料カルテは、体質・気血水・天気相性・今日/明日の予報を入口として見せる。
Plusは、Plus深掘りチェックを加えて、以下を保存版として整理する。

- 私の未病トリセツカード
- 不調ループの核
- 強くなる前の前触れサイン
- 重ねすぎ注意カード
- からだの負担ラインとツボ候補
- 季節と天気を先回りする守り方
- 今日・明日の予報ページの使いどころ
- 専門家に相談するときの共有メモ

## 予報ページとの差別化

予報ページは、その日の気象データを使った「今日・明日の作戦表」。
Plusは、ユーザーがなぜ崩れやすいのかを理解する「保存版の地図」。

そのため、Plusでは日々のケア提案を置き換えない。
具体的な当日のツボ・食養生・方針は予報ページに寄せ、Plusでは以下を扱う。

- どんな条件が重なると崩れやすいか
- どんな前触れがあるか
- どんな習慣・飲食の組み合わせを避けたいか
- どの身体ラインが関係しやすいか
- どんなタイミングで予報ページを見るとよいか
- 相談時に何を伝えるとよいか

## 同梱ツール

### 重ねすぎ注意カード

Plus深掘りチェックの回答から、避けたい習慣・飲食の組み合わせをカード化する。
ユーザーを責める表現にはせず、「警戒日前に1つだけ外す負担」として見せる。

### からだの負担ラインとツボ候補

今日/明日の予報ページではM-test系のツボ提案を1点に絞る。
Plusでは、ユーザーに関係しやすい身体ラインのツボ候補を保存版カードとして見せる。

当日実際に使うツボは、気象データも反映する予報ページを優先する。

## 注意

7方針は内部ロジック・編集材料として使うが、Plus本文で「固定の優先順位」を主役にしない。
日々の予報ページに出る7つの習慣/方針と矛盾して見えないようにする。
