# 体調予報 V2 仕様（2026-07-21）

## 目的

V1の「気圧低下／上昇など6方向 × 固定された体質親和性」を、次の役割分担へ変更する。

1. 気象イベントが、身体を揺さぶる強さを決める。
2. アクセル／ブレーキの連続値が、揺れの表れ方を決める。
3. 余力の連続値が、同じ気象イベントでの崩れやすさを小さく補正する。
4. 気・血・水の6スコアが、張り・重さ・乾き・消耗などの内訳とケア方針を決める。

この予報はセルフケアのための仮説であり、診断や発症の断定には使用しない。

## 気象イベント

スコアでは、相関する方向を足し合わせず、次の3グループから最大1要因ずつ採用する。

- 気圧：`pressure_shift`を1回だけ計上。上昇／低下／往復は表示と小さな体質補正に使う。
- 気温：`temperature_shift`、絶対的な`cold`、`heat`のうち最大1つを計上する。
- 水分環境：`damp`、`dry`のうち最大1つを計上する。

気温の通常の日内変化は割り引く。たとえば34℃から28℃への夕方の低下は「気温差」にはなり得るが、「冷え込み」にはしない。

湿気・乾燥は露点から算出した絶対湿度を主に使い、相対湿度は補助情報とする。気温上昇だけで相対湿度が下がった場合を、強い乾燥として扱わない。

## 体質とスコア

- 気象負荷は全員に共通する割合を62%、個人差を38%とする。
- 気圧の向きとアクセル／ブレーキが一致した場合の増幅は最大6%に制限する。反対方向への減点はしない。
- 余力補正は0.94〜1.08に制限する。
- モード判定は丸め前の精密スコアを使う。
  - 0.0〜3.9：安定
  - 4.0〜6.9：いたわり
  - 7.0〜10.0：守り
- DBの`score_0_10`は互換性のため整数のまま保存し、小数は`score_display_0_10`と`score_precise_0_10`へ保存する。

## 生成時刻

- 01:05 JST：今日と明日のスナップショットを生成する。今日分は日中固定する。
- 16:30 JST：明日分だけを再生成する。今日分には触れない。
- 17:00 JST：更新済みの明日分で夜通知する。16:30更新がない場合だけ、通知処理が明日分を補完する。
- ページ表示だけでは、予報ロジックのバージョン違いを理由に自動再生成しない。

## 保存する追跡情報

`vendor_meta`と`computed.radar_plan_meta.risk_context`に、次を保存する。

- 予報モデル・気象抽出モデルのバージョン
- 朝／夕方／通知補完の生成スロット
- 体質・地域のフィンガープリント
- 3気象グループの負荷と重複抑制結果
- 連続体質軸、気血水6スコア、余力補正
- スコアの寄与内訳とモード判定前の精密値

過去の記録に保存された予報スナップショットは再計算しない。

## ロールバック

通常は`RADAR_FORECAST_MODEL_VERSION=v2`を使用する。緊急時は環境変数を`v1`へ変更すると、気象抽出とパーソナライズの両方を旧モデルへ戻せる。

## 回帰テスト

`tests/radar-forecast-v2.test.mjs`で、気圧上昇／低下の対称性、往復時の二重加点防止、夏の気温低下、相対湿度の見かけの低下、両体質の両方向反応、余力補正、精密スコア境界、生成時刻、ケア方針文言を固定している。
