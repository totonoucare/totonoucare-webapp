# v7.78.21 天気負荷カードの情報整理

## 目的

天気負荷カードで、気象のピーク時刻と体調不良が出る時刻を同一視させず、一日に何の負荷がどの程度あるかを優先して伝える。

## 3カード

表示順は次の3層に統一する。

1. 気温／湿度／気圧
2. 高温、気温上昇、湿気、乾燥、気圧低下、上下変動などの現象
3. 個人補正後の負荷 高／中／低

各カード内には時刻を置かない。計算済みのピーク時刻と方向データは削除せず、代表ピーク選択に利用する。

## 天気負荷のピーク

- 3群のうち、個人負荷が中以上（`load >= 0.34`）で、有効な開始・終了時刻を持つ要素だけを候補とする
- 候補の個人負荷が最大の1件だけを、3カード下のピルへ表示する
- 表示するのは気象現象のピークであり、症状発現の予測時刻ではない
- 条件を満たす要素がない日は、ピル自体を表示しない

## 絶対気温注意

猛暑・酷暑・厳しい寒さは、体質別ゆらぎ度と別の情報である。

- ゲージ上部には置かない
- 天気負荷枠の下へ置く
- 初期状態はタイトルだけの閉じた`details`
- 開いた時だけ詳細、公的警戒アラートではない旨を表示する

## ケアとの役割分担

予報ヒーロー内の`注意時間の前に／先回りケア`カードは削除する。暮らす・食べる・ほぐす等の具体策は、正式な対策ケアに一本化する。

## 変更しないもの

- 体調ゆらぎ度
- 3群の天気負荷
- 体質親和性と余力補正
- 気象ピークの算出・保存
- 猛暑・厳寒注意の判定条件
- 今日01:05固定、明日夕方の再計算
- V1ロールバック
- DB、SQL、環境変数
